# 红黑博弈 · 信任与背叛的心理课（课堂互动网页版）

把课件《红黑博弈：信任与背叛的心理课》变成一堂**全班 50 人都能参与**的沉浸式卡牌博弈。
纯静态网页（HTML + 原生 JS，零依赖、可离线），适合投到教室大屏，也适合直接部署到 GitHub Pages。

> 设计严格遵循课件四段流程：**破冰与规则 → 博弈与观察（暗战 5 局 → 谈判 5 局）→ 解析与反思 → 迁移与收束**。

## 二、在教室里怎么用（50 人参与方案）

1. 老师打开 `index.html`（投影到大屏）。
2. **课前设置**：把 50 人分成若干小组（默认 **5 组 × 10 人**，可在 2–12 组间调整），填上组名，确认「暗战 5 局 / 谈判 5 局」。
3. 点「开始上课」。全班进入博弈：
   - **暗战（信息封闭）**：每组围在一起秘密商量，统一出「红牌」或「黑牌」；老师在大屏上点对应组的红/黑按钮。
   - **谈判（开放沟通）**：各组代表可下座位串访、口头承诺「我们都出黑」，用大屏右上角「外交时间」倒计时控制沟通时长，再统一出牌。
4. 每轮点「🔓 揭晓本轮」——大屏立刻显示各组出牌、本轮得失与累计比分，并弹出结果横幅（全黑共赢 / 单红独吞 / 多红互伤 / 全军覆没）。

> 🔒 **出牌保密**：在揭晓前，大屏上每家只显示中性「✓ 已出牌」标记，**不显示红/黑颜色**，小组之间互相看不到别人的牌；只有点「揭晓本轮」后全班才看到各家出牌。这正好还原课件里「暗战（信息封闭）」的设定。
5. 10 轮结束点「结束并进入反思」，进入 **解析与反思**：展示最终排名 + 课件里的信任成本、生活副本、合作清单，学生可填写自己的「可信承诺」与「合作建议」（仅保存在本人设备，可截图分享）。

**50 人如何“都参与”**：每组内部必须讨论、协商、达成一致才能出牌——这 50 人全都卷入了「合作 vs 背叛」的真实抉择，正是红黑博弈的精髓。代表串访阶段还能全员走动沟通。

## 三、得分规则（源自课件第 5 页）

设本轮出红牌的组数为 `r`，总组数为 `n`：

| 情形 | 条件 | 每组得分 |
|------|------|----------|
| **全黑共赢** | `r = 0`（全黑） | 每组 **+3** |
| **单红独吞** | `r = 1`（仅 1 组红） | 红组 **+5**，黑组 **0** |
| **多红互伤** | `r ≥ 2` 且仍有黑组 | 红组 **+1**，黑组 **+3** |
| **全军覆没** | `r = n`（全红） | 每组 **−1** |

## 四、本地预览（不上网也能跑）

直接用浏览器打开 `index.html` 即可。或起一个本地服务器（手机也能连同一 WiFi 访问）：

```bash
# 在文件所在目录执行
python -m http.server 8000
# 浏览器访问 http://localhost:8000
```

## 五、部署到 GitHub Pages（你自己操作，AI 不推送）

> 按我们约定，文件只改在本地，由你本人上传 / 部署。步骤如下：

1. 登录 GitHub（账号 `xuchengyuan123`），新建一个仓库，例如 `redblack-game`。
2. 把本目录里的 **`index.html`、`course.html`、`README.md`** 三个文件上传到仓库根目录（可直接拖拽，或用 Web 端 Upload files）。
3. 进入仓库 **Settings → Pages**，Source 选 **Deploy from a branch**，分支选 **main**（或 master），目录选 **/ (root)**，保存。
4. 等待一两分钟，访问 `https://<你的用户名>.github.io/redblack-game/` 即可。全班用手机/电脑打开这个链接就能看讲义；大屏游戏由老师在本机打开运行。

> 提示：GitHub Pages 是静态托管，**不需要实名认证**，适合你直接使用。
